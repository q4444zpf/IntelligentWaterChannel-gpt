# 智能水槽系统界面设计总结

## 1. 项目概述

智能水槽系统面向智慧水利实验、明满流混合实验、水槽运行监测、闸门控制、水泵调节、历史数据分析和报警处理等场景。系统通过三维数字孪生工艺图，将水槽结构、实时数据、设备状态、控制操作、趋势曲线、历史数据和报警信息统一集成到 Web 平台中。

系统核心页面包括：

1. 实时监控页面
2. 历史数据页面
3. 报警数据页面

三个页面共用统一的顶部综合导航状态栏，并通过菜单进行切换。

---

## 2. 整体设计风格

系统整体采用深色科技风、数字孪生大屏风格和工业监控平台风格。

设计关键词：

- 深蓝 / 黑蓝主色调
- 蓝色发光描边
- 半透明面板
- 工业 SCADA 风格
- 三维数字孪生风格
- 高保真数据驾驶舱
- 绿色 / 黄色 / 红色状态标识
- 表格、曲线、三维模型联动

状态颜色建议：

| 状态 | 颜色 | 说明 |
|---|---|---|
| 在线 / 正常 | 绿色 | 设备正常运行 |
| 预警 / 注意 | 黄色 / 橙色 | 数据接近阈值或存在轻微异常 |
| 报警 / 异常 | 红色 | 超限、故障、反馈异常等 |
| 离线 | 灰色 | 设备无连接或无数据 |
| 控制中 | 蓝色 | 正在执行控制指令 |

## 4. 实时监控页面设计

## 4.1 页面定位

实时监控页面用于展示水槽系统的当前运行状态，包括三维工艺图、闸门状态、传感器状态、设备控制、实时报警和趋势曲线。

主要目标：

- 三维展示智能水槽工艺图；
- 实时显示闸前水位、闸后水位、闸门开度、泵频率等数据；
- 显示设备在线、离线、异常状态；
- 支持闸门开度、水泵频率手动控制；
- 显示实时报警；
- 通过曲线图查看实时趋势。

---

实时监控页面采用三列布局：

- 左列：状态总览
- 中列：三维工艺图 + 实时趋势分析
- 右列：手动控制 + 报警信息

┌────────────────────────────────────────────────────────────────────────────┐

# 5. 历史数据页面设计

## 5.1 Tab1：历史数据查询与曲线分析

用于根据时间段、设备类型、设备名称查询历史数据，同时显示曲线变化图表和明细表格，并支持导出 CSV。

约束条件：

- 设备类型只能单选；
- 设备名称可以多选；
- 查询结果显示表格和曲线图；
- 支持 CSV 导出。

- 结束时间

- 数据状态

设备类型：水位计
设备名称：WL-01、WL-02、WL-03
所属渠道：全部

## 5.2 Tab2：传感器时序总览与节点水位回放

### 5.2.1 页面定位

用于根据时间段展示各时间点传感器数据总览，并通过空间剖面曲线图和时间轴查看各节点水位变化。

核心变化：

- 曲线图 X 轴不再按时间显示；
- X 轴改为按渠道划分的水位计和闸门位置显示；
- 下方配时间轴，用于切换不同时间点；
- 当前时间点变化时，空间剖面曲线随之更新。

- 第一列时间固定；
- 支持横向滚动；
- 当前时间轴对应行高亮；
- 列头显示设备名称、所属渠道、数据项、单位；
- 空值显示为 `--`；
- 支持分页；
- 支持导出 CSV。

# 7. 页面间联动设计

三个页面之间需要支持业务联动。

## 7.1 实时监控 → 历史数据

场景：点击传感器或趋势图中的“查看历史”。

跳转效果：

- 进入历史数据页面；
- 自动选中对应设备；
- 默认查询近 1 小时；
- 自动加载曲线和表格。

## 7.2 实时监控 → 报警数据

场景：点击右侧报警信息中的“查看更多报警”。

跳转效果：

- 进入报警数据页面；
- 默认筛选今日未处理报警。

## 7.3 报警数据 → 实时监控

场景：点击报警列表中的“定位”。

跳转效果：

- 进入实时监控页面；
- 三维模型聚焦报警设备；
- 报警设备红色闪烁。

## 7.4 报警数据 → 历史数据

场景：点击报警列表中的“曲线”。

跳转效果：

- 进入历史数据页面；
- 自动选择报警设备；
- 时间范围设置为报警前后 10 分钟；
- 曲线图标记报警点。

# 10. 总结

智能水槽系统界面设计最终形成三个核心页面：

```text
智能水槽系统
├── 实时监控
│   ├── 三维水槽工艺图
│   ├── 闸门实时状态
│   ├── 传感器实时状态
│   ├── 手动控制设备
│   ├── 实时报警信息
│   └── 实时趋势分析
├── 历史数据
│   ├── 历史数据查询与曲线分析
│   └── 传感器时序总览与节点水位回放
└── 报警数据
    ├── 报警查询
    ├── 报警统计
    ├── 报警列表
    ├── 报警处理
    └── 报警联动
```

其中：

- 实时监控页面解决“当前系统运行状态怎么看、设备怎么控制”的问题；
- 历史数据页面解决“历史数据怎么查、曲线怎么分析、时序怎么回放”的问题；
- 报警数据页面解决“报警怎么查、怎么处理、怎么定位和复盘”的问题。

整体形成完整闭环：

```text
实时监控发现问题
↓
报警数据记录和处理问题
↓
历史数据复盘问题过程
↓
导出数据用于实验分析和报告编制
```
