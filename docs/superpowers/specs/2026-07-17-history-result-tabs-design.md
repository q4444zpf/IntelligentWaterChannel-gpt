# 历史结果双 Tab 设计

## 目标

将历史数据页中纵向排列的“历史曲线分析”和“历史数据表格”合并为一个结果面板，通过 Tab 切换视图，缩短页面并提高曲线与表格之间的切换效率。

## 界面结构

- 查询区保持不变。
- 查询区下方只保留一个历史结果面板。
- 面板顶部显示两个 Tab：`历史曲线（设备数）` 和 `数据表格（记录数）`。
- 默认选中“历史曲线”。
- “历史曲线”内容继续使用现有的单设备独立图表网格。
- “数据表格”内容继续使用现有表格字段、空状态和记录数说明。
- 两个视图不再各自使用独立的大面板标题，避免重复边框和纵向留白。

## 交互规则

- Tab 切换只改变结果展示，不触发查询，也不改变时间、设备、粒度或状态筛选条件。
- 查询、刷新和重置完成后保留当前结果 Tab。
- 用户离开历史数据主页面再返回时，结果 Tab 恢复为“历史曲线”。
- CSV 导出仍导出完整查询结果，与当前打开的结果 Tab 无关。
- 没有查询结果时，曲线 Tab 显示现有曲线空状态，表格 Tab 显示现有表格空状态。

## 图表生命周期

- ECharts 组件只在“历史曲线”Tab 激活时挂载。
- 从表格切回曲线时按当前查询结果重新挂载图表，确保画布宽高有效。
- Tab 切换不重新生成历史数据。

## 视觉要求

- Tab 使用现有深色工业监控风格，选中态使用蓝色边框和背景。
- Tab 标题、数量和焦点状态清晰，支持键盘聚焦与点击。
- Tab 内容区域宽度、三列/两列/单列响应式规则保持不变。
- 切换时不出现内容重叠、横向滚动或明显布局跳动。

## 验收标准

1. 查询区下方只显示一个结果面板。
2. 默认打开“历史曲线”Tab。
3. 用户可在曲线和表格之间切换，筛选条件和查询结果不变。
4. Tab 标题正确显示当前设备数和记录数。
5. 查询、刷新、重置与 CSV 导出行为保持正常。
6. 离开历史数据页面再返回时默认回到“历史曲线”。
7. 浏览器控制台无 ECharts 隐藏容器尺寸警告。
8. 桌面与窄屏视口下无文字、控件或内容重叠。
