# 实时趋势单设备简化设计

## 目标

简化“实时趋势分析”的设备选择与节点水位表达：节点水位保留全节点剖面但不再使用沿程距离；流量计、水位计和倒虹吸改为图表上方下拉单选；泵设备只保留 P1 且不显示选择控件。

## 节点水位变化

- 保留“节点水位变化”Tab 和当前实测/模拟双曲线。
- 节点仍按工程拓扑顺序排列：前池、G0 闸后、渠①、G1 闸前/后、渠②，依次到渠⑥和集水池。
- X 轴由数值距离轴改为类别轴，类别值为节点名称；节点之间等距展示。
- 数据适配层不再向图表提供业务距离，排序只依赖显式工程顺序。
- 删除 X 轴“沿程距离 L（m）”名称、距离刻度和底部 DataZoom 滑块。
- 保留 Tooltip、实测/模拟图例、G0–G6 标识、渠段背景、上下限、刷新、暂停与复位。
- 复位只恢复图例和图表交互状态，不再重置距离缩放。

## 时间趋势设备选择

流量计、水位计和倒虹吸压力 Tab 使用图表上方紧凑下拉框，一次只显示一个设备。下拉选项按工程顺序排列，显示设备编号和位置。切换设备后立即刷新曲线，并在该 Tab 内保存当前选择。

泵频率设备目录只保留 P1。泵频率 Tab 固定展示 P1，不显示设备下拉，但在图表工具栏明确显示“设备：P1 一号变频泵”。

删除以下多选能力：

- 左侧设备筛选栏。
- 搜索和区域筛选。
- Checkbox、多选、全选/取消全选。
- 已选设备标签。
- 多设备叠加曲线。

时间范围、5 秒刷新、暂停/继续、立即刷新、Tooltip、缩放、框选、统计和错误状态保持不变。

## 组件与状态

新增 `TrendDeviceSelect.vue`，只负责顶部设备下拉：

```ts
props: {
  devices: readonly TrendDevice[];
  modelValue: string;
}
emits: {
  'update:modelValue': [deviceId: string];
}
```

`TrendAnalysis.vue` 删除 `DeviceSelector.vue`，在流量、水位和倒虹吸 Tab 中组合 `TrendDeviceSelect + TimeRangePicker + TrendChart`。泵 Tab 只显示 P1 文本。

控制器的每个 Tab 偏好从 `selectedIds: string[]` 收敛为 `selectedDeviceId: string`。数据源查询契约保留 `deviceIds` 数组以兼容后续接口，但控制器始终传入单元素数组。空设备只在目录配置错误时出现，并显示空状态。

`DeviceSelector.vue` 若无其他引用则删除。通用 `TrendChart.vue` 保持接收 `TrendSnapshot`，但每次只有一条系列。

## 数据配置

- 流量计保留 FM-01、FM-02、FM-03。
- 水位计保留 WL-01 至 WL-06。
- 泵设备仅保留 P1，删除模拟 P2、P3。
- 倒虹吸保留 SI-01 至 SI-04，仍只展示压力 MPa。
- 默认设备为各目录工程顺序中的第一个在线设备。

## 布局

- 时间趋势不再使用左右分栏，图表占模块完整宽度。
- 图表上方工具栏从左到右为设备信息/选择、时间范围、统计摘要。
- 模块标题行继续放 Tab、5 秒状态、暂停、刷新和复位。
- 1024×900 下工具栏允许分成两行，下拉框和时间控件不得覆盖图表。

## 测试与验收

自动化测试更新为：

1. 泵目录只有 P1。
2. 控制器为每个 Tab 独立保存一个 `selectedDeviceId`。
3. 查询始终只包含一个设备 ID。
4. 节点水位 option 使用 category X 轴，不包含 DataZoom，不出现距离轴标题。
5. 节点类别与数据点按相同工程顺序排列。
6. 时间趋势 option 每次只有一个系列，并保留正确单位、Tooltip、框选与缩放。
7. 全量 Node 测试、`vue-tsc --noEmit` 和 Vite 构建通过。

浏览器验收：

1. 节点水位无底部距离滑块，节点按顺序等距显示。
2. 流量、水位和倒虹吸下拉可切换设备，图表立即更新为对应单条曲线。
3. 泵 Tab 只显示 P1 且没有设备下拉。
4. 所有时间趋势图占满模块主体宽度。
5. 桌面与 1024×900 下无溢出、重叠和 ECharts 尺寸警告。
