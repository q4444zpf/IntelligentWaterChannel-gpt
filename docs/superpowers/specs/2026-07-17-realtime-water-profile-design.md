# 实时节点水位沿程曲线设计

## 目标

参考 `I:\大水槽\设计图` 项目中 `MonitorPage.vue` 使用的 `WaterDepthProfile.vue`，在当前项目“实时监控 > 实时趋势分析 > 节点水位曲线”中集成沿程节点水位剖面图。保留参考图的空间剖面表达和核心视觉，同时使用当前项目已有 ECharts、组件规范和实时数据结构。

## 现状对比

参考项目使用 Vue 3、Element Plus 和 ECharts 6。实际“水深曲线”组件是自绘 SVG `WaterDepthProfile.vue`，包含实测/模拟双曲线、距离轴、水深轴、G0–G6 闸门标线和渠段分区；数据在组件内静态定义。

当前项目使用 Vue 3 和 ECharts 5，实时趋势仍由 `canvas-charts.js` 手写 Canvas 绘制。当前水位数据分散在 `monitoring-data.js` 的 `GATES` 与 `SENSOR_GROUPS` 中，尚未提供实时接口。

## 技术方案

- 使用当前已有 ECharts 5 构建沿程剖面，不引入新依赖。
- 新建 `RealtimeWaterProfileChart.vue`，负责图表初始化、更新、Tooltip、缩放、响应式尺寸和销毁。
- 新建 `useRealtimeWaterProfile.js`，负责数据加载、5 秒轮询、暂停、继续、立即刷新、错误和最后更新时间。
- 新建纯数据适配模块，将现有闸前、闸后和水位计数据转换为沿程节点数组，并提供默认模拟数据源。
- 当前模拟数据源通过 composable 参数注入；后续接入接口时只替换数据源函数，不修改图表组件。

## 数据模型

统一快照结构：

```js
{
  timestamp: 1784250000000,
  nodes: [
    {
      id: 'WL-01',
      label: '前池',
      distance: 0,
      measured: 0.402,
      simulated: 0.396,
      state: '在线'
    }
  ]
}
```

- `distance` 是水槽沿程距离，范围 `0–830 m`。
- `measured` 使用当前水位计和闸门前后水位。
- `simulated` 使用可重复的平滑偏差模拟参考曲线。
- 离线节点使用 `null`，曲线保留断点，Tooltip 显示无数据。
- 节点按水槽前池到集水池的工艺顺序排列。

## 图表语义与视觉

- 标题语义为“节点水位曲线”。
- X 轴为“沿程距离 L（m）”，Y 轴为“节点水位 H（m）”。
- 实测水位使用青色发光实线和圆形节点。
- 模拟水位使用蓝色虚线。
- 图例显示“实测水位”和“模拟水位”，可点击显隐。
- Tooltip 显示时间、节点名称、距离、实测水位、模拟水位和状态。
- 使用 `markLine` 显示 G0–G6 闸门位置。
- 使用分区背景、底部标签或等效 ECharts 图形标识前池、渠①–渠⑥和集水池。
- 提供上下限参考线，沿用当前实时趋势的 `0.50 m` 上限和 `0.20 m` 下限。
- 图表支持内部缩放和平移、恢复完整范围和 ResizeObserver 自适应。
- 颜色、边框、文字和背景适配当前深色工业监控风格，不复制 Element Plus 样式。

## 实时状态与操作

- 首次进入节点水位 Tab 显示加载状态。
- 默认每 5 秒获取一个新快照。
- “暂停”停止轮询但保留当前图表；“继续”恢复轮询。
- “立即刷新”即时加载，不重置下一次轮询能力。
- “复位”恢复完整缩放范围，不修改数据。
- 刷新期间显示轻量更新状态，不遮挡已有图表。
- 刷新失败时保留上一帧有效数据并显示错误提示。
- 没有任何有效节点时显示空数据状态，不绘制误导性零值。
- 组件卸载时清除定时器和 ResizeObserver。

## 与现有趋势模块集成

- `RealtimeView.vue` 增加真实的趋势 Tab 状态。
- 默认 Tab 仍为“节点水位曲线”。
- 节点水位 Tab 渲染新 ECharts 剖面组件。
- 其他趋势 Tab 继续渲染现有 Canvas，原有标签和时间范围工具按钮保留。
- 节点水位 Tab 工具栏显示 `5秒刷新`、`暂停/继续`、`立即刷新` 和 `复位`。
- Tab 切换不影响三维监控、控制面板、报警摘要或页面整体尺寸。

## 响应式与可访问性

- 图表容器使用稳定高度和宽度，ResizeObserver 触发 ECharts resize。
- 宽屏显示完整闸门与渠段标签；窄屏减少坐标刻度密度并避免 Tooltip 超出容器。
- 图表提供描述性 `aria-label`。
- 状态按钮使用文字和现有按钮样式，禁用态明确。
- 文字、图例、闸门标线和 Tooltip 不相互遮挡。

## 验收标准

1. 默认节点水位 Tab 显示沿程水位剖面，不再显示旧节点 Canvas 曲线。
2. 实测/模拟曲线、闸门标线、渠段、坐标轴、单位、图例和 Tooltip 语义正确。
3. 初始数据来自当前 `GATES` 与 `SENSOR_GROUPS`，节点按工艺顺序排列。
4. 每 5 秒数据发生可见更新，暂停、继续、立即刷新和复位可用。
5. 加载、空数据和刷新错误状态均有明确展示。
6. 切换到其他趋势 Tab 时继续显示原有 Canvas 内容和工具按钮。
7. 1920x1080 与窄屏视口下图表非空、标签可读且页面无新增溢出。
8. 浏览器控制台无错误、ECharts 零尺寸警告或未清理定时器行为。
9. `npm test` 和 `npm run build` 通过。
