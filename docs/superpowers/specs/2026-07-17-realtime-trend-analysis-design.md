# 实时趋势分析工作台设计

## 目标与范围

完善实时监控页面中部下方的“实时趋势分析”模块，以五个 Tab 分别承载节点水位、流量计、水位计、泵频率和倒虹吸压力分析。节点水位保留现有沿程剖面；其余四类设备使用统一的时间趋势工作台，支持多设备选择、筛选、时间范围、实时刷新和 ECharts 交互。

本次不改变三维工艺监控、左右设备状态、手动控制和报警区域。不引入 Ant Design Vue，继续使用当前项目的 SCADA 样式与 ECharts 5，避免引入另一套 UI 基线。

## 现状与约束

- `RealtimeView.vue` 当前直接管理趋势 Tab、节点水位轮询和旧 Canvas 图表。
- 节点水位已实现沿程距离剖面，包含实测/模拟水位、G0–G6、渠段、阈值、缩放和 5 秒刷新。
- 其余四个 Tab 仍复用同一张旧 Canvas 图，没有设备筛选或真实 Tab 语义。
- 当前数据只有 1 个流量计、6 个水位计，泵和倒虹吸目录不完整。
- 项目以 JavaScript 为主。本次新增趋势模块使用 TypeScript，但不迁移无关现有文件。

## 总体架构

`RealtimeView.vue` 只组合页面区域，并以 `<TrendAnalysis />` 代替当前内联趋势模块。趋势工作台拆分如下：

```text
src/
├── components/realtime/trend/
│   ├── TrendAnalysis.vue
│   ├── TrendTabs.vue
│   ├── DeviceSelector.vue
│   ├── TrendChart.vue
│   └── TimeRangePicker.vue
├── composables/
│   └── useRealtimeTrends.ts
├── config/
│   └── trendConfig.ts
└── data/
    └── realtime-trend-data.ts
```

- `TrendAnalysis.vue`：管理当前 Tab，组合筛选器、时间选择和图表，协调节点剖面与时间趋势模式。
- `TrendTabs.vue`：只负责五个 Tab 的呈现与切换，通过 props/emits 通信。
- `DeviceSelector.vue`：负责搜索、区域/渠道筛选、全选/取消、复选列表和已选标签。
- `TrendChart.vue`：通用 ECharts 时间折线图，负责 option、Tooltip、图例、缩放、框选、恢复和 ResizeObserver 生命周期。
- `TimeRangePicker.vue`：最近10分钟、最近1小时、最近24小时和自定义起止时间。
- `useRealtimeTrends.ts`：拥有选择状态、查询状态、5 秒轮询、暂停/继续、错误保留和派生统计。
- `trendConfig.ts`：类型定义和四种时间趋势配置。新增设备类型只需扩展配置和数据目录。
- `realtime-trend-data.ts`：模拟目录、确定性时间序列和可替换的异步数据源接口。

新增 `typescript`、`vue-tsc` 与 `tsconfig.json`，并增加类型检查脚本。现有 JavaScript 文件通过 `allowJs` 参与项目但不要求本次补齐历史类型。

## Tab 行为

### 节点水位变化

- 完全保留当前沿程剖面，X 轴为沿程距离，Y 轴为节点水位。
- 工艺顺序继续为前池、渠①至渠⑥、集水池。
- 不增加时间滑块，不显示左侧设备筛选器或通用时间范围选择。
- 保留 5 秒刷新、暂停/继续、立即刷新、复位、实测/模拟图例、闸门和渠段。

### 流量计趋势分析

- 设备目录包含 FM-01、FM-02、FM-03，按工程位置排序。
- X 轴为时间，Y 轴为流量，目录允许 L/s 或 m³/s；同一当前模拟目录统一使用 L/s，避免同图混合单位。
- 支持名称搜索、多选、全选/取消、标签移除和图例显隐。
- Tooltip 显示时间、设备名称、位置和流量值。

### 水位计趋势分析

- 使用 WL-01 至 WL-06，按渠道顺序排列。
- 支持渠道/区域筛选、搜索、多选和图例显隐。
- X 轴为时间，Y 轴为水位 m。
- 每台已选设备显示当前值、最高值和最低值。

### 泵频率趋势分析

- 模拟目录包含 P1、P2、P3。
- X 轴为时间，Y 轴为频率 Hz。
- 显示每台泵的当前、最大、最小频率和当前启停状态。
- 图表用背景区段标识停止时段，Tooltip 同时显示频率和运行/停止状态。

### 倒虹吸趋势分析

- 模拟目录包含 SI-01 至 SI-04。
- 仅展示压力，不混合流量和水位。
- X 轴为时间，Y 轴为压力 MPa。
- 支持名称搜索、多选、全选/取消、标签移除和图例显隐。

## 配置与数据模型

核心类型：

```ts
type TrendType = 'flow' | 'level' | 'pump' | 'siphon';
type TimeRangeKey = '10m' | '1h' | '24h' | 'custom';

interface TrendDevice {
  id: string;
  name: string;
  type: TrendType;
  location: string;
  region: string;
  unit: 'L/s' | 'm' | 'Hz' | 'MPa';
  state: '在线' | '异常' | '离线';
  color: string;
}

interface TrendPoint {
  timestamp: number;
  value: number | null;
  running?: boolean;
}

interface TrendSeries {
  device: TrendDevice;
  points: TrendPoint[];
}

interface TrendSnapshot {
  timestamp: number;
  series: TrendSeries[];
}
```

数据源契约：

```ts
queryTrendData({
  trendType,
  deviceIds,
  startTime,
  endTime,
  now
}): Promise<TrendSnapshot>
```

模拟目录为 3 个流量计、6 个水位计、3 台泵和 4 个倒虹吸。模拟值使用设备 ID、时间戳和趋势类型产生确定性波动，便于测试和刷新验证。离线读数为 `null`，图表显示断点而不是零值。

采样规则：

- 最近10分钟：10秒一个点，共约61点。
- 最近1小时：1分钟一个点，共约61点。
- 最近24小时：20分钟一个点，共约73点。
- 自定义：根据跨度计算粒度，限制为最多约120点。

## 选择与时间交互

- 每个时间趋势 Tab 独立保存选择和时间范围，切换 Tab 后恢复原状态。
- 默认选择该类型所有在线设备，并按工程顺序显示；离线设备仍在列表中并带状态标识。
- 搜索和区域筛选只改变可见候选，不自动移除已选设备。
- 全选只选择当前筛选结果；取消全选只移除当前筛选结果。
- 已选标签按工程顺序显示，点击标签上的移除按钮取消选择。
- 选择为空时不发起查询，显示“请至少选择一个设备”。
- 自定义时间要求开始时间早于结束时间，非法范围在控件内提示。
- 最近10分钟、最近1小时和最近24小时在轮询时让窗口随当前时间前移；自定义范围保持固定起止时间并重新查询同一区间。

## 刷新与状态

- 时间趋势默认每5秒刷新，节点水位继续使用现有独立轮询。
- 暂停会清除轮询但保留当前曲线；继续会立即刷新并恢复单一轮询。
- 同一 Tab 的并发刷新被抑制。
- 首次请求显示加载状态；已有数据刷新时只显示轻量“更新中”。
- 请求失败保留上一帧有效数据，并显示错误提示。
- 没有有效数据时显示空状态，不绘制误导性零线。
- Tab 或页面卸载时清理计时器、ResizeObserver 和 ECharts 实例。

## 图表规范

- 使用 ECharts Line Chart，每台设备使用稳定、可辨识的颜色和图例项。
- X 轴为时间，Tooltip 使用轴触发并限制在容器内。
- Y 轴名称和单位来自配置；当前每个 Tab 只有一种量纲，使用单 Y 轴。
- 支持滚轮缩放、拖动、底部 DataZoom、框选缩放、恢复视图和图例开关。
- 曲线连接策略为不连接 `null`，异常和离线状态可在 Tooltip 中识别。
- 水位与泵频率在图表上方显示统计摘要；泵停止区段使用克制的半透明背景。
- 图表组件暴露 `resetZoom()`，不向父组件暴露 ECharts 实例。

## 布局与视觉

- 保持当前深色工业 SCADA 色彩、边框、按钮和文字层级。
- 顶部显示模块标题、五个 Tab 和刷新状态。
- 时间趋势主体左侧设备筛选约占24%，右侧时间控件、统计和图表约占76%。
- 左侧面板是工具区域，不再套装饰卡片；图表占右侧主要空间。
- 已选设备使用紧凑标签，移除使用熟悉的关闭图标并提供标题说明。
- 在中央区域变窄时，标题与操作区分行，设备栏收窄，时间操作允许有序换行；图表保持至少180px有效高度。
- 所有按钮、标签和 Tooltip 不得溢出或遮挡相邻信息。
