# 历史数据多设备独立曲线设计

## 目标

完善历史数据查询与曲线分析页面。用户按时间范围直接选择多个设备，查询后每台设备显示一张独立时间曲线图，并按设备在智能水槽工艺中的前后位置排列。桌面端每行显示三张图。

## 范围

本次改动仅涉及“历史数据 > 历史数据查询与曲线分析”标签页，包括查询条件、设备选择、曲线结果和历史数据表格联动。实时监控、节点水位回放和报警页面不调整。

## 查询区

- 保留开始时间、结束时间、数据粒度和数据状态。
- 去掉设备类型筛选。
- 使用一个设备多选控件直接展示全部设备。
- 设备选项同时显示设备编号、类型和所属位置，例如 `WL-01 | 水位计 | 渠①`。
- 设备选项按水槽工艺的前后位置排列；同一位置内按设备编号自然排序。
- 多选控件支持关键词搜索、逐项勾选、全选和清空。
- 已选设备以可移除标签展示。
- 默认选择项目已有历史示例数据的设备，并按工艺顺序显示。
- 保留查询、重置、刷新和导出 CSV 操作。

## 曲线区

- 每台已选设备独占一张曲线图，不将多个设备合并到同一画布，也不使用双 Y 轴。
- 查询结果按设备在水槽工艺中的前后位置排列，不按勾选先后顺序排列。
- 宽屏桌面端使用三列网格，超过三台设备后自动换行。
- 较窄屏幕降为两列或单列，确保坐标轴、标题和工具不重叠。
- 每张图的标题显示设备编号、设备类型、所属位置、指标和单位。
- X 轴统一为查询时间范围内的时间。
- Y 轴按设备类型定义：
  - 水位计：水位，单位 `m`。
  - 闸门：开度，单位 `%`。
  - 流量计：流量，单位 `L/s`。
  - 压力计：压力，单位 `MPa`。
  - 水泵：频率，单位 `Hz`。
- 每张图独立提供悬停数据提示、区域缩放、平移和重置视图能力。
- 图表显示查询时间范围、数据点数量、最新值和设备状态。
- 无数据的设备仍保留对应图位，并显示明确的无数据状态。

## 数据与排序模型

前端示例数据使用统一设备结构：设备编号、类型、所属位置、工艺顺序、指标、单位、状态和时间序列。工艺顺序是曲线卡片与设备选择列表的唯一默认排序依据。

查询动作根据已选设备和时间范围生成当前结果集。图表网格和历史数据表格读取同一结果集，避免两处设备范围不一致。重置恢复默认时间范围、默认设备和默认粒度；刷新保留当前条件并重新生成结果。

## 历史表格与导出

- 历史数据表格仅显示当前时间范围和已选设备的数据。
- 表格保留时间、设备类型、设备名称、所属渠道、数据项、数值、单位和状态字段。
- 表格数据按时间倒序，再按设备工艺顺序排列。
- CSV 导出使用与表格相同的数据范围和字段，并正确转义中文及逗号内容。
- 未选择设备或没有结果时禁用导出并显示空状态。

## 状态与错误处理

- 未选择设备时，查询按钮不可执行，并提示至少选择一台设备。
- 开始时间晚于结束时间时显示时间范围错误，不更新当前结果。
- 查询期间显示加载状态并避免重复提交。
- 无匹配数据时显示空状态，不绘制误导性的零值曲线。
- 离线或异常设备使用状态标识，但仍允许查看已有历史数据。

## 技术方案

使用 Apache ECharts 渲染独立曲线图。每个设备图表封装为可复用 Vue 组件，负责初始化、更新、尺寸响应和销毁。历史页负责查询条件、设备选择、排序、结果生成、表格与 CSV 导出。

不继续扩展现有手写 Canvas 历史图，因为缩放、平移、提示和多个独立实例会显著增加自研交互复杂度。实时页已有 Canvas 图保持不变。

## 验收标准

1. 历史查询区不再显示设备类型筛选。
2. 用户可直接搜索并多选不同类型设备，也可全选或清空。
3. 查询后每台设备只出现在自己的曲线图中。
4. 桌面端曲线按工艺前后顺序每行三列显示，超过三台自动换行。
5. 水位计和闸门分别显示正确的 Y 轴指标与单位，其他设备类型同样正确。
6. 每张图可悬停查看数据，并可独立缩放、平移和重置。
7. 重置、刷新和 CSV 导出与当前查询条件一致。
8. 历史表格只显示当前已选设备和时间范围的数据。
9. 页面构建成功，常用桌面与窄屏尺寸下无文字或控件重叠。
